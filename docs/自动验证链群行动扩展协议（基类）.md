# 自动验证链群行动扩展协议（基类）

## 1. 关于协议

此扩展协议，基于 LOVE20 扩展协议标准，扩展行动参与机制，允许行动者以链群的方式参与行动。

- 行动者可自由加入退出所选链群，进行可自动验证的共识行动
- 行动者基于自动验证结果获得行动铸币激励

## 2. 初始参数

### 2.1 最小治理票占比

持有大于等于 最小治理票占比 的治理者，可创建链群成为链群服务者

### 2.2 容量倍数

- 单个链群最小容量：已铸造代币总量 \* 最小治理票占比 \* 容量倍数
- 链群服务者链群理论容量上限：已铸造代币总量 \* 链群服务者治理票占比 \* 容量倍数

### 2.3 质押代币合约地址

### 2.4 质押倍数

- 质押量 = 链群容量 / 质押倍数
- 最小质押量 = 单个链群最小容量 / 质押倍数
- 最大质押量 = 链群服务者链群理论容量上限 / 质押倍数

### 2.5 最大参与代币倍数

- 单个行动者最大参与代币数 = 已铸造代币总量 / 最大参与代币倍数

### 2.6 最小参与代币量

单个行动者参与行动时，最少需要提供的代币数量

## 3. 链群服务者

任何治理者可以在[链群](https://love20tkm.github.io/group/)里铸造自己的链群 NFT 成为链群服务者

### 3.1 链群激活或恢复

治理者可以选择自己持有的链群 NFT 来激活一个新链群或恢复旧链群，激活时需提供：

- 链群说明：例如：一些注意事项，以及如何联系到链群服务者，如何加入所在链群的行动者群（例如：QQ 群、微信验证群）等
- 行动者最小参与量
- 行动者最大参与量（可选）：0 为不做限制

链群激活时，需无偿质押代币，决定链群的容量上限。

链群容量 = 累计质押量 \* 质押倍数

### 3.2 链群更新

立即更新，并在当轮立即生效（不影响已参与的行动者）

- 链群说明
- 行动者最小参与量
- 行动者最大参与量

### 3.3 链群扩容

链群服务者可追加质押，增加链群容量上限。行动者加入链群时，如果超出链群容量，将无法加入。

### 3.4 链群自动化验证

子类实现每轮的自动化验证，决定每个行动者的铸币量。

每轮的验证阶段，下面 6 种交互情况任意情况发生，立即快照并生成链群的所有行动者的验证结果

- 行动者加入行动
- 行动者退出行动
- 链群激活
- 链群取消激活
- 铸造上轮行动激励
- 主动触发

注意：

- 链群服务者链群理论容量上限必须要大于等于已激活的链群的总容量，否则将无法生成快照与验证结果
- 若当前轮次链群未触发自动验证，则自动放弃本链群所有激励，分给其他链群

### 3.5 链群取消激活

链群服务者在链群创建后的下一轮开始，可随时取消激活链群，并立即返还质押代币。

链群取消激活后，行动者可继续领取之前未领取的行动铸币激励。

已取消激活的链群，新行动者无法加入，现有行动者也无法追加代币；链群服务者也无法进行链群更新和链群扩容。

## 4. 行动者

### 4.1 加入行动

行动者参与行动时，需指定一个链群 ID，参与代币数需满足：

- 不低于初始参数中的最小参与代币量
- 不超过单个行动者最大参与代币数

### 4.2 铸造行动激励

行动者加入并经过验证阶段后，可铸造自己的行动激励

子类实现：行动者铸造行动激励

链群理论铸币激励 = 链群内所有行动者铸造行动激励

因为是自动化验证，故：链群实际铸币激励 = 链群理论铸币激励

### 4.3 退出行动

可随时退出并立即返还累计参与的所有代币
