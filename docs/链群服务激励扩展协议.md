# 链群服务激励扩展协议（初稿）

## 1. 关于协议

此扩展协议，基于 LOVE20 扩展协议标准，扩展行动参与机制，对链群行动中的链群服务者做激励

- 链上参与：参与链群服务激励行动
- 自动计算：链上自动计算铸币激励
- 激励分配：支持设置多个接收地址和分配比例

## 2. 初始参数

- 代币地址
- 链群行动 id

## 3. 激励铸造

链群服务者实际铸币激励 = 链群服务者激励行动总激励 \* 链群服务者服务的所有链群实际铸币激励 / 链群行动激励

## 4. 激励分配

链群服务者实际铸币激励可以预先设置二次分配

### 4.1 分配设置

链群服务者可以设置多个接收地址和分配基点（basis points）：

- 基点范围：0-10000（10000 = 100%）
- 基点总和不能超过 10000
- 最多设置 10 个接收地址
- 剩余部分（10000 - 基点总和）归原始账户

### 4.2 历史记录

- 每次设置/修改分配时，记录当前轮次的分配设置
- 领取激励时，使用该轮次生效的分配设置（向前查找最近的设置）

### 4.3 分配计算

```
某接收地址的激励 = 总激励 × 该地址基点 / 10000
原始账户的激励 = 总激励 - 所有接收地址激励之和
```
